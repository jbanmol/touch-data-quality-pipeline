Touchdata_id Generation Process ▼

Each touch sequence gets a unique identifier (Touchdata_id) that groups related touch points:

Touchdata_id = hash(fingerId + startTime + sessionId)
Creates unique identifiers for each continuous touch sequence

Why Traditional ML Over Neural Networks?

Interpretability

Traditional ML methods provide clear explanations for their decisions. We can understand exactly why a touch sequence was flagged as anomalous or rated with a specific quality score.

No Training Data Required

Unsupervised methods work without labeled examples. The system learns patterns directly from the data without needing human-annotated "good" vs "bad" examples.

Computational Efficiency

Statistical methods and traditional ML algorithms are fast and lightweight, making them perfect for real-time analysis and batch processing of large datasets.

Robustness

Traditional methods are less prone to overfitting and work well with smaller datasets. They provide consistent results across different data distributions.

Clustering Algorithms for Behavioral Classification ▼

Clustering helps identify different types of touch behaviors automatically:

K-Means Clustering

Groups touch sequences into 4 behavioral types based on timing and movement patterns. Like sorting people into personality types based on their behavior.

DBSCAN

Finds dense groups of similar behaviors and identifies outliers. Good at finding unusual interaction patterns that don't fit common categories.

Gaussian Mixture Models

Assumes behaviors follow bell-curve distributions. Provides probability scores for how well each sequence fits different behavioral patterns.

Hierarchical Clustering

Builds a tree of behavioral similarities. Helps understand relationships between different interaction styles.

Spatial_Efficiency = euclidean_distance(start, end) / total_path_length

2. Interaction Type: Behavioral Classification ▼

The interaction type describes how the user behaved during the touch sequence. It's like identifying personality types based on movement patterns.

Precise

Careful, deliberate movements with consistent timing. Users who take their time and move smoothly. Often indicates focused attention and careful execution.

Quick

Fast, efficient movements with short duration. Users who know what they want and execute quickly. Common in experienced users or simple tasks.

Hesitant

Slow, uncertain movements with pauses. Users who are thinking or unsure. Often seen in complex tasks or when users are learning.

Erratic

Irregular, unpredictable movements. Could indicate difficulty, distraction, or technical issues. Requires careful interpretation.

Valid Touch Phase Sequences ▼

Every valid touch sequence must follow this pattern:

# Valid sequence patterns VALID_PATTERNS = [ "Began → Ended", # Simple tap "Began → Moved → Ended", # Basic swipe "Began → Stationary → Ended", # Press and hold "Began → Moved → Stationary → Ended", # Complex gesture "Began → Moved → Canceled", # Interrupted gesture "Began → Stationary → Canceled" # Canceled hold ] # Invalid patterns (will be flagged) INVALID_PATTERNS = [ "Moved → Began", # Wrong order "Ended → Moved", # Continues after end "Began → Began", # Duplicate start "Moved" (without Began), # Missing start "Began" (without Ended/Canceled) # Missing end ]

Why These Rules Matter

  • Data Integrity: Ensures touch sequences represent complete interactions
  • Research Validity: Incomplete sequences can skew timing and behavior analysis
  • System Reliability: Helps identify data collection or processing issues
  • Comparative Analysis: Standardized patterns enable cross-study comparisons

Touchdata_id-Based Validation

Modern Validation Approach ▼

The system uses Touchdata_id to group related touch points and validate complete sequences. This is more reliable than the legacy seqId approach because it handles complex interactions better.

# How Touchdata_id validation works def validate_touch_sequence(df, touchdata_id): # Get all points for this sequence sequence = df[df['Touchdata_id'] == touchdata_id] # Sort by event_index to get correct order sequence = sequence.sort_values('event_index') # Extract the phase pattern phases = sequence['touchPhase'].tolist() # Check against valid patterns if is_valid_pattern(phases): return 'VALID' else: return get_specific_flag(phases) def is_valid_pattern(phases): # Must start with Began if phases[0] != 'Began': return False # Must end with Ended or Canceled if phases[-1] not in ['Ended', 'Canceled']: return False # Middle phases must be Moved or Stationary for phase in phases[1:-1]: if phase not in ['Moved', 'Stationary']: return False return True

Validation Flag Types

SEQUENCE_INCOMPLETE

Missing required start (Began) or end (Ended/Canceled) phases. Like a sentence without a period - technically incomplete and potentially misleading for analysis.

INVALID_PHASE_ORDER

Touch phases appear in wrong order (e.g., Ended before Began). Indicates data collection issues or processing errors that need investigation.

DUPLICATE_PHASES

Same phase appears multiple times inappropriately (e.g., two Began phases). Suggests system glitches or data corruption.

TIMING_ANOMALY

Unusual timing patterns like negative time differences or impossibly fast movements. Could indicate clock synchronization issues.
